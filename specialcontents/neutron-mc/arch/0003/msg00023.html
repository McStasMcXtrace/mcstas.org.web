I have adopted a solution for the numerous output parameters I had in my components : I put all of them into structures. Thus there is only one symbol than contains all the data. Then, the Monitor_Optimizer uses the same principle as your Adapt_check for getting the information : MC_GETPAR with the optimizer name from the instrument definition. This adds an input parameter to Monitor_Optimizer. I updated this fact in the doc attached.
