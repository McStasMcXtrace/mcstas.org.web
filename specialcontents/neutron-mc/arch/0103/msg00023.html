I don't think that this problem would be solved by a compiler initializing
variables to zero. The variable has to be reset every time a neutron
enters this component. Since such a component is part of a main program, a
compiler could only set it once (for the very first neutron) to zero,
which is not enough. This variable has to be set ot zero for all neutrons
being tracked in the component.
